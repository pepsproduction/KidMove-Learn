# 🏃‍♂️🎉 KidMove Learn - สวนผลไม้นับเลขแสนสนุก

**KidMove Learn** คือเว็บแอปพลิเคชันเกมการศึกษาที่ควบคุมด้วยการเคลื่อนไหวของร่างกาย (Motion-Controlled Web Game) ออกแบบเป็นพิเศษสำหรับเด็กปฐมวัยอายุ 4–6 ขวบ (อนุบาล 1–3) เพื่อช่วยส่งเสริมพัฒนาการด้านกล้ามเนื้อมัดใหญ่ การประสานงานระหว่างตา-มือ-ร่างกาย และทักษะพื้นฐานทางคณิตศาสตร์ (การนับเลขและเปรียบเทียบจำนวน 1–10) ผ่านการจำลอง "สวนผลไม้นับเลข"

เกมนี้ประมวลผลการตรวจจับท่าทางในเครื่องของผู้ใช้ทั้งหมด (Client-Side Processing) มีความปลอดภัยสูงต่อความเป็นส่วนตัวของเด็ก (Privacy-First) และออกแบบให้เป็น Static Site ที่สามารถนำไป Deploy บน **GitHub Pages** ได้ฟรีทันที

---

## 🎨 การออกแบบ UI/UX สำหรับเด็กปฐมวัย

*   **สีสันสดใสแบบพรีเมียม (Rich Aesthetics):** ใช้โทนสีอ่อนโยน เช่น เขียวชอุ่ม (Grass Green) และฟ้าสดใส (Sky Blue) ร่วมกับปุ่มทรงกลมมนและฟอนต์ขนาดใหญ่พิเศษ (Outfit & Inter) อ่านง่าย สดใส ไม่สร้างความระคายเคืองตา
*   **ตัวละครและผลไม้ขนาดใหญ่:** ออกแบบให้มีขนาดใหญ่พอเหมาะสำหรับการเล่นผ่านจอโทรทัศน์ (TV) หรือโปรเจคเตอร์ (Projector) ภายในห้องเรียน
*   **ภาษาทางบวก (Positive Reinforcement):** หลีกเลี่ยงคำว่า "แพ้", "ผิด", "ไม่ได้" โดยเมื่อนับเลขเกินระบบจะแจ้งอย่างอ่อนโยนว่า *"ลองใหม่นะ อีกนิดเดียวจ้า"*
*   **ระบบเสียงนำทางภาษาไทย (Speech Synthesis):** นำทางเด็ก ๆ ด้วยเสียงอ่านโจทย์วิเคราะห์ภาษาไทยผ่านลำโพงโดยตรง (TTS) ช่วยพัฒนาทักษะการฟัง

## ⚡ เทคโนโลยีที่เลือกใช้

1.  **Vite:** เครื่องมือจัดการโมเดลและ Build ระบบที่รวดเร็วสูง
2.  **Vanilla CSS & HTML5 Canvas:** เพื่อประสิทธิภาพการเรนเดอร์เฟรมเกม 60 FPS ปราศจากความล่าช้า
3.  **MediaPipe Tasks Vision (Pose Landmarker):** ตรวจจับท่าทางโครงร่างร่างกาย 33 จุดผ่านเว็บแคมโดยตรง โดยการโหลดโมเดลแบบน้ำหนักเบา (Lite Model) และตัวประมวลผล WebAssembly (WASM) จาก CDN เพื่อหลีกเลี่ยงขนาดแพ็กเกจที่ใหญ่เกินไป
4.  **Web Audio API (Synthesized Audio):** สังเคราะห์ซาวด์เอฟเฟกต์ (ชิปจูน) ขึ้นมาเองแบบเรียลไทม์ ทำให้เกมเล่นเสียงได้ทันทีโดยไม่ต้องโหลดไฟล์เสียง `.mp3`
5.  **Web Speech Synthesis API:** สำหรับขับเสียงอ่านโจทย์ภาษาไทย
6.  **LocalStorage:** เก็บคะแนนประวัติการเล่นของคุณครูและผู้เรียนแบบออฟไลน์

## 🛠️ วิธีการรันโปรเจกต์บนเครื่องของคุณ (Local Setup)

    ตรวจสอบให้แน่ใจว่าติดตั้ง Node.js บนเครื่องของคุณเรียบร้อยแล้ว


## 🚀 วิธีการ Deploy ขึ้น GitHub Pages

เนื่องจากไม่มี Backend ทำให้แอปนี้ทำงานในรูปแบบ Static Site 100% จึงเหมาะสำหรับการ Deploy ไปยัง GitHub Pages:

### วิธีที่ 1: ใช้ gh-pages npm package (แนะนำสำหรับนักพัฒนา)

1.  ติดตั้งตัวช่วยติดตั้ง:
    ```bash
    npm install -D gh-pages
    ```
2.  เพิ่มสคริปต์ใน `package.json`:
    ```json
    "scripts": {
      "dev": "vite",
      "build": "vite build",
      "preview": "vite preview",
      "deploy": "vite build && gh-pages -d dist"
    }
    ```
3.  ตรวจสอบตัวเลือก `base` ใน `vite.config.js` ให้ตรงกับชื่อโฟลเดอร์ Repository บน GitHub ของคุณ (เช่น `base: '/KidMove-Learn/'`)
4.  รันคำสั่งเพื่อ Deploy:
    ```bash
    npm run deploy
    ```
    *ระบบจะสร้างโฟลเดอร์ `dist` และส่งขึ้นหน้าเว็บ `gh-pages` ของ GitHub อัตโนมัติ*

### วิธีที่ 2: ใช้ GitHub Actions (อัตโนมัติเมื่อกด Push)

สร้างไฟล์ `.github/workflows/deploy.yml` แล้วใช้ GitHub Pages Static Action เพื่อจับผลลัพธ์จากโฟลเดอร์ `dist` ไปจัดทำ Deployment เมื่อมีการ Push โค้ดเข้าสาขา `main`

## 🎮 รายการฟีเจอร์การตรวจจับร่างกายและท่าทาง (Gestures ใน MVP)

1.  **เอียงตัวซ้าย / เอียงตัวขวา (Lean Left/Right):** ใช้เอียงเพื่อเลื่อนตะกร้าไปทางซ้าย/ขวาอย่างนุ่มนวล โดยอ้างอิงจากระยะเอียงของหัวไหล่เทียบกับตอนบันทึก Baseline เริ่มต้น
2.  **ยกมือขวาขึ้นสูง (Raise Right Hand):** ยกมือขวาพ้นแนวระดับหัวไหล่เพื่อตบ/เก็บผลไม้ที่ตกมาอยู่ในจุดครอบครองของตะกร้า
3.  **ยกสองมือขึ้นพร้อมกัน (Raise Both Hands):** ยกสองมือขึ้นสูงในระยะ Calibration ค้างไว้ 1.5 วินาทีเพื่อยืนยันเริ่มเกม
4.  **ระบบสำรองเมื่อไม่มีกล้อง (Keyboard Fallback):** หากไม่ได้ให้สิทธิ์กล้อง
    *   ใช้แป้นลูกศร `⬅` และ `➡️` เลื่อนตะกร้าซ้าย-ขวา
    *   ใช้ปุ่ม `Enter` แทนการยกมือเพื่อตบเก็บผลไม้
    *   คุณครูสามารถควบคุมผ่านแผงคอนโซลกลางเพื่อช่วยเหลือเด็ก ๆ ได้

## 📊 รายการหัวข้อทดสอบการทำงานของระบบ (QA Checklist)

*   [x] **NPM Build:** รันผ่านเรียบร้อยไม่มีข้อผิดพลาด (`ROLldown/Vite Build Success`)
*   [x] **Camera Handshake:** ขออนุญาตกล้องและแสดงรูปภาพเป็นกระจกกลับด้านเพื่อความคุ้นเคยของเด็ก
*   [x] **Camera Refusal:** ปฏิเสธกล้อง แล้วตัวเกมสลับมาใช้แป้นพิมพ์ช่วยเหลือแทนได้ทันทีโดยไม่ค้าง
*   [x] **Calibration Stars:** แสดงตัวอย่างร่างกาย ให้เด็กยืนและตรวจจับความสมบูรณ์ หากกล้องไม่เห็นส่วนลำตัวจะเตือนเด็กให้ขยับถอยหลังเป็นภาษาไทย
*   [x] **Math random targets:** สุ่มตัวเลขโจทย์ (1-3, 1-5, 1-10) และสุ่มจับคู่ผลไม้หลักพร้อมตัวหลอกตกมาสลับสับเปลี่ยน
*   [x] **Interactive Collection:** ตะกร้าเลื่อนตามไหล่เอียงชนกับผลไม้เป้าหมาย และต้องยกมือขวาจึงจะเก็บคะแนนได้ หากปล่อยผ่านจะไม่สูญเสียสิทธิ์
*   [x] **Teacher real-time skip/reset/pause:** สั่งการข้ามวิชา รีเซ็ตบอร์ด ปรับความง่าย/ยากได้แบบทันควันผ่านเมนูควบคุม

## 🔮 ข้อจำกัดและแนวทางต่อยอด

### ข้อจำกัดในปัจจุบัน (MVP):
*   ตัวจับโมเดล MediaPipe ต้องการแสงสว่างที่เพียงพอในห้องเรียนเพื่อให้วิเคราะห์จุดพิกัดหัวไหล่และข้อมือได้แม่นยำ
*   เกมเล่นคนเดียวในเวอร์ชัน MVP เพื่อควบคุมความสับสนของกล้อง

### โครงสร้างการเชื่อมต่อวิชาในอนาคต (Architecture Extensibility):
วิชาภาษาไทย (เช่น การเก็บตัวอักษรเรียงคำ ก-ฮ) และวิชาภาษาอังกฤษ ได้ทำการเตรียมไฟล์ placeholder ไว้ที่ `src/games/thai/placeholder.js` และ `src/games/english/placeholder.js`
เมื่อระบบเกมต้องการขยายวิชา สามารถเปลี่ยน Subject Config จาก `SUBJECTS.MATH` ไปเป็นวิชาอื่น ๆ ได้โดยดึงคำถามมาวาดบน Canvas และสลับท่าทางเงื่อนไขให้เหมาะกับการสะกดคำได้ทันที
